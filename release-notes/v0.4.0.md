# TimesBar v0.4.0

One theme this release: **you no longer need the web UI to fix a forgotten timer**. Log past sessions, edit the running timer's begin/project/activity in place, all from the menu bar — with a mouse-friendly drag strip that shows what's already logged.

## New

### Log a past entry
StartTimerForm gains a **Log past entry** toggle. Flip it on and the form expands with:
- A **Recent** list of your last 5 project + activity + note combos — one click prefills the form (does *not* submit) so you only need to set the time.
- The same custom **calendar grid** used by Time off — 30pt day cells, no cramped popovers.
- A 12h **time-range bar** for begin + end. On today: window is `now − 10h … now + 2h`, with a live "now" marker. On other days: `09:00 … 21:00`. Drag handles to resize, drag the band middle to translate, ±1min nudge buttons below for fine-tuning. Snaps to 5-minute boundaries.
- A **Set end time** toggle — when off, submit creates a *running* timer backdated to the chosen begin (e.g. "I started 40 minutes ago and forgot to hit play").

### Edit the running timer
Pencil button on the active timer section opens a new edit form (`PATCH /api/timesheets/{id}`):
- Change begin via the drag strip (single-handle mode — band fades to "now" with a gradient).
- Change project / activity / note in place — no restart, no duplicate entry; the elapsed counter keeps ticking against the (possibly new) begin.
- Hint copy under the form adapts to which fields changed.

### Drag-bar visualizes what's already logged
The time-range bar overlays every timesheet inside its window as muted gray blocks behind your selection:
- **Handles snap at neighbor boundaries** — drag begin left toward a logged session and it stops at that session's end; same for the end handle dragging right into the next one.
- **Band translate can fly over blocks** during the drag (tints red while parked over an overlap) and snaps to the nearest free slot of the same duration on release. If no slot fits anywhere in the window, the band reverts to its pre-drag position.
- Hover any gray block for a tooltip with `HH:mm — HH:mm  description`.

### Cursor hints
Resize cursor over handles, open-hand over the band, closed-hand while dragging. Hovering off the bar resets to the default arrow.

### Wider dropdown when the forms are open
`MenuBarExtra` auto-widens to 420pt for the past-entry / edit-active forms and shrinks back to 320pt for the main view, Time off, and Monthly balance.

## Under the hood

- `KimaiClient.createTimesheet` now accepts an optional `end` (was required) — same call powers the CSV backfill, the "log completed past entry", and the "backdated running timer" paths.
- New `KimaiClient.updateTimesheet(id:project?:activity?:begin?:end?:description?:)` wraps `PATCH /api/timesheets/{id}`. Only fields that changed are sent.
- `CalendarPicker` extracted out of `TimeOffView` so the new forms can reuse `DateFieldRow` + `CalendarGrid` + `CalendarDayCell`.
- New `TimeRangeBar` component with no-overlap clamping (`leftBlocker`/`rightBlocker`), band-translate validation (`overlapsAnyExisting` + `nearestFreeSlot`), and an existing-entries overlay sourced from `TimerStore.nearbyEntries`.
- `TimerStore` gains `logEntry(...)`, `updateActiveTimer(...)`, `refreshNearbyEntries(around:)`, and `clearNearbyEntries()`.
- `MenuBarView` adds a route-aware `dropdownWidth` so width tracks the active panel.

## Install

```bash
brew install xcodegen
git clone https://github.com/nangert/TimesBar.git
cd TimesBar
make install
open /Applications/TimesBar.app
```

Or grab the prebuilt `TimesBar-v0.4.0.zip` from the release assets and right-click → **Open** in `/Applications` to bypass Gatekeeper on first launch.
