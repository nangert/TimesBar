# TimesBar v0.5.0

Ten QoL features in one release. Theme: **catch forgotten timers, fix entries without the web UI, and customize the app to your install.**

## New

### Catching forgotten timers

- **Sleep-wake reconciliation.** When the Mac wakes after ≥10 min of sleep with a timer still running, a card appears at the top of the dropdown with three options: *Keep elapsed time*, *Backdate stop to before sleep*, or *Split into a new entry at wake* (preserves the old entry up to sleep, starts a fresh running timer at wake with the same project/activity/description/tags). No setting — always on.
- **End-of-day auto-stop.** Settings → Behavior → "Auto-stop timer at end of day" + a time picker (default 19:00). If a timer is still running past that wall-clock time, TimesBar PATCHes `end = the configured time` (not "now") so the Kimai record is accurate. A toast pops with an **Undo** button for 30 seconds; Undo calls `/restart?copy=all` to start a fresh continuation entry.
- **Idle detection.** Settings → Behavior → "Prompt when idle" + threshold slider (5–60 min, default 15). When there's no keyboard/mouse input past the threshold with a timer running, a card asks whether to keep elapsed time or backdate the stop to when you went idle. Fires once per idle session; resets when the user becomes active again.

### Fixing entries from the menu bar

- **Right-click any recent row** → context menu with **Edit / Duplicate / Delete**. Edit opens a new full timesheet form with editable date, project, activity, description, tags, and a draggable time-range bar (existing entries shown as colored blocks, no-overlap clamping). Duplicate POSTs to `/timesheets/{id}/duplicate`. Delete asks for confirmation before DELETEing.
- **Tags everywhere.** All forms (start, edit running, edit any, log past) now have a Tags field with chip-style multi-select and autocomplete from `/api/tags`. Tag chips show on quick-start rows + the past-entry suggestions list.
- **Right-click the menu bar icon** → native menu with *Start last activity* / *Stop timer* without opening the dropdown. (Skipped *Log past entry* for v1 — `MenuBarExtra` has no API to programmatically open the dropdown from a Carbon callback.)
- **Global hotkey ⌘⌥T.** Settings → Behavior → toggle on; pressing the combo system-wide toggles the timer (stops the running one, or restarts your most recent if idle). No key recorder UI yet — fixed combo for v1.

### Customization

- **Configurable Kimai base URL.** Settings (gear) → new **Server** section. Paste your Kimai instance URL, hit Save; the client rebuilds and the next API call uses it. Inline validation rejects non-`http(s)` schemes. The README's hardcoded-URL apology is finally gone.
- **Today's hours line.** Below the week bar chart: `5h 12m today · 2h 48m to target` (switches to `· over target` once you exceed). Ticks live every second while a timer runs.
- **Color-coded projects.** Each project gets a stable color (sourced from Kimai's `color` field if set; deterministic Knuth-hash-derived HSL otherwise, with red and kimai-green hue bands skipped to keep state UI distinct). Colors appear on the week bar chart (stacked segments per project per day), quick-start row dots, time-range bar existing-entry overlays, and the running-timer indicator.

## Under the hood

- New `UserPreferences` singleton (`UserDefaults.standard` wrapper) underpins all new settings.
- New `SettingsView` replaces the bare `TokenSetupForm` for the gear-icon route — composes Token, Server, and Behavior sections.
- New `TimeRangeBar` overlay shows existing logged entries (gray blocks → colored blocks in v0.5.0) and clamps handle drags at neighbor boundaries.
- New observers: `SleepObserver` (NSWorkspace willSleep/didWake), `IdleMonitor` (CGEventSource polling every 30 s), `HotkeyManager` (Carbon `RegisterEventHotKey`).
- New endpoint wrappers: `KimaiClient.tags()`, `deleteTimesheet(id:)`, `duplicateTimesheet(id:)`. `createTimesheet` and `updateTimesheet` now accept optional `tags: [String]?`.
- Comprehensive sign-out + auth-reject teardown — Carbon hotkey unregisters, sleep snapshot clears, ephemeral overlay state resets.
- 61 new unit tests (112 total) covering today-hours math, idle-edge detection, project color hashing + hue-band avoidance, auto-stop time arithmetic, sleep reconciliation thresholds, and tag normalization.

## Install

```bash
brew install xcodegen
git clone https://github.com/nangert/TimesBar.git
cd TimesBar
make install
open /Applications/TimesBar.app
```

Or grab the prebuilt `TimesBar-v0.5.0.zip` from the release assets and right-click → **Open** in `/Applications` to bypass Gatekeeper on first launch.
