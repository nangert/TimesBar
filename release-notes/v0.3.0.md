# TimesBar v0.3.0

Two big themes since v0.2.0: **everything auto-configures from your Kimai profile**, and **you can manage time off without leaving the menu bar**.

## New

### Inline time-off requests
Open Time off → click **+** next to *Upcoming* → an inline form drops in:
- Type picker (Urlaub / Zeitausgleich / Krankheit / Elternzeit / Unbezahlter Urlaub / Sonstiges)
- Date picker with a **custom calendar grid** — 30pt-tall day cells, hover highlight, kimaiGreen selection, a ring on today, dimmed prev/next-month days. SwiftUI's `.compact` and `.graphical` styles are both designed for trackpads at ~20–24pt cells; this grid is laid out for a mouse.
- Multi-day toggle that reveals an End date picker (auto-clamps "To" ≥ "From")
- Half-day toggle (hidden when multi-day is on)
- Optional note

Hover any *Upcoming* row → a **×** button reveals → one click cancels the request.

### Auto-configuration from `/api/users/me`
Zero manual settings. TimesBar reads:
- `hours_per_week` → Today target (e.g. ~7.7h for a 38.5h week, no more hardcoded 8h)
- `holidays` → annual vacation budget
- `work_start_day` → contract start; first year is prorated automatically (a July start gets 12.5/25 days)
- `public_holiday_group` → which country's holidays drive Monthly balance (without it, `/api/public-holidays` returns empty on most installs)

### Per-year vacation breakdown
Time off now mirrors the Kimai dashboard exactly. Each tracked year shows `used / available` with a "prorated" badge on the contract-start year, the current year bolded, and a cumulative remaining total across every year.

### Quick-start preserves description + tags
Tapping a quick-start row now hits `PATCH /api/timesheets/{id}/restart` with `copy=all` instead of POSTing a fresh entry, so the original description + tags carry over to the new timer.

### TimeBuzzer CSV import tool
New `timesbar-import` CLI for backfilling historical CSV data:
```bash
make import-tool
build/timesbar-import --list-projects
build/timesbar-import --csv FILE                    # prints a mapping template
build/timesbar-import --csv FILE --mapping MAP.json --dry-run
```
Dedupes against existing entries by (begin-minute, project) so re-runs are safe. Handles CRLF line endings and BOM-prefixed exports. Diagnostic commands: `--list-holidays YEAR`, `--show-user`.

## Fixes

- **Token revocation routes back to sign-in.** A revoked or expired API token previously showed stale data forever; `KimaiClient` now parses 401/403 into a typed `KimaiError.unauthorized` and the menu bar flips back to the token form.
- **Pagination instead of silent truncation.** `KimaiClient.timesheets()` previously asked for `size=500` and dropped anything beyond. Now pages through `page=1,2,…` until the server returns a short page (or 404). The importer's dedup pre-fetch and yearly data loads can no longer miss entries.
- **DST-safe day math.** Monthly balance used `+86_400` seconds to advance "to tomorrow", which silently shifted by an hour twice a year on DST days. Now uses `Calendar.date(byAdding: .day)`.
- **Monthly balance hides empty months** unless they're the current month — viewing a past year no longer shows "0/0" rows for months you didn't log in.
- **Quick-start failures surface inline.** A failed quick-start (e.g. activity no longer linked to project) now shows a red error line below the rows instead of failing silently.
- **`TokenStore.save()` checks Keychain result.** A failing Keychain write no longer leaves the app claiming "authenticated" with nothing actually stored.
- **Test parallelism race fixed.** `KimaiClientTests` wrapped in `@Suite(.serialized)` so parallel test runs can't clobber `MockURLProtocol.handler`.

## Under the hood

- 24 new unit tests covering vacation proration, monthly balance math, pagination, and 401/403 handling. Total: 51 tests.
- `MenuBarView` routes via an enum instead of four `@State` Bools.
- Removed dead code: `Sparkline.swift`, `LaunchAtLogin.requiresApproval`.

## Install

```bash
brew install xcodegen
git clone https://github.com/nangert/TimesBar.git
cd TimesBar
make install
open /Applications/TimesBar.app
```

Or grab the prebuilt `TimesBar-v0.3.0.zip` from the release assets and right-click → **Open** in `/Applications` to bypass Gatekeeper on first launch.
