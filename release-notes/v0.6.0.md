# TimesBar v0.6.0

Visual polish for the dropdown plus a round of internal cleanup. Theme: **the same app, smoother to look at and half as much copy-pasted code underneath.**

## New

### Visual polish

- **Animated progress bars.** The Today bar and the vacation card now share one capsule-capped `ThinProgressBar` with smooth width animation. Once you pass the daily target, the Today fill shifts to a green→mint gradient so "done" reads at a glance.
- **Rolling elapsed counter.** The active timer's `HH:MM:SS` readout rolls its digits (`contentTransition(.numericText())`) instead of jumping.
- **Quick-start hover affordance.** Hovering a recent row fades in a green ▶ next to the duration and shows a "Restart this entry" tooltip — the click-to-restart action is no longer invisible.
- **Week chart tooltips + weekend colors.** Hover any bar for `Wed · 7h 30m` (or `Sat · no time logged`). Weekend bars with logged hours now render their per-project color stack; previously they were always washed-out gray even when you'd worked.
- **Tags autocomplete highlight.** Suggestion rows in the tags dropdown highlight on hover (the old handler was a no-op).
- **Honest sign-in copy.** The token hint now names the host from your configured base URL instead of a hardcoded instance.

## Under the hood

- **TimerStore on a diet.** Nine mutating methods shared one copy-pasted guard/refresh/catch block — now a single `perform()` helper. `signOut()` and `handleUnauthorized()` build on a shared `resetLiveSessionState()` (their hand-maintained reset lists had drifted). Vacation accounting moved to `TimerStore+Vacation.swift`. 1,043 → 912 lines with identical behavior.
- **Edit forms share their guts.** New `TimesheetFieldsSection` renders the project/activity/note/tags rows once for all three forms; `sortedProjects`/`sortedActivities` live on the store. The 6× `initial*` snapshot pattern became `TimesheetDraft`, which owns the PATCH-diff rules (trimmed descriptions, order-insensitive tags, 1-second begin/end tolerance, minimal payloads) — previously untested view code, now covered by 6 unit tests.
- **Component dedup.** `PanelHeader`/`CloseButton` (six hand-rolled headers), `TagChipsRow` (two tag-pill copies), `promptCardStyle()` + `TimesheetContextSummary` (sleep/idle/auto-stop cards). Close buttons gained a "Close" tooltip.
- **Formatter hygiene.** One shared `HH:mm` formatter replaces five per-render `DateFormatter()` allocations; `UserMe.workStartDate` (hit repeatedly via the vacation-breakdown chains), `MonthlyBalanceView`, and `TimeOffView` use static formatters.
- **Build reliability.** `make` now regenerates the Xcode project on every build, so newly added source files can't silently miss a stale `.xcodeproj`.
- 6 new unit tests (118 total).

## Install

```bash
brew install xcodegen
git clone https://github.com/nangert/TimesBar.git
cd TimesBar
make install
open /Applications/TimesBar.app
```

Or grab the prebuilt `TimesBar-v0.6.0.zip` from the release assets and right-click → **Open** in `/Applications` to bypass Gatekeeper on first launch.
