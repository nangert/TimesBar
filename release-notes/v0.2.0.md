# TimesBar v0.2.0

Big polish pass since v0.1.0 — two new pages, no more modal sheets, and Launch-at-login.

## New

### Inline forms instead of sheets
The **Settings** panel and the **+ New timer** picker now flip the dropdown body in place instead of opening a separate sheet. The sheet-based versions hit a SwiftUI quirk where dismissing a sheet inside a `MenuBarExtra` dropdown tangled with the dropdown's own close, requiring three cancels before the state finally cleared. Inline transitions sidestep that completely.

### Time off page
Reachable via the footer ellipsis menu (**…** → **Time off…**):
- **Urlaub** card with a progress bar against your annual × years-tracked budget.
- **Upcoming** approved absences with type-colored dots (Urlaub / Krankheit / Freizeitausgleich / Andere).
- Annual budget is the only number you set. The tracking start year is **auto-detected** from your earliest timesheet via `GET /api/timesheets?orderBy=begin&order=ASC&size=1`. Used days are summed from approved holiday absences in that range.

### Monthly balance page
**…** → **Monthly balance…** opens a year-by-month view with a `‹ year ›` switcher (defaults to the current year):
- For each month: `actual / expected` hours plus a colored chip with the running balance — kimai-green when you're at or above target, soft red when under.
- Expected hours assume **8h/day, Mon–Fri**, with public holidays (`GET /api/public-holidays`) and approved absences (`GET /api/absences`) reducing expected proportionally to their `halfDay` flag.
- Current month is clipped to today so the chip reflects "where you're at right now", not "where you'll be on the 31st".
- A `YEAR` total row at the bottom sums everything.

### Launch at login
New **Launch at login** toggle in the footer ellipsis menu. Uses `SMAppService.mainApp` — also visible under System Settings → General → Login Items.

## Fixes

- Active timer is no longer blank when one is actually running. Kimai's `/api/timesheets/active` returns `project` / `activity` as nested objects, not bare IDs — the decoder now accepts both shapes.
- Starting a timer and filtering by date range now work. Kimai's API expects HTML5-local datetime (`yyyy-MM-dd'T'HH:mm:ss`, no timezone), not full ISO 8601 with `Z`.
- Elapsed counter keeps ticking while the dropdown is open. Timers are now scheduled on `.common` run-loop mode so they don't freeze in `eventTracking`.
- Menu bar label always renders. The previous compound `HStack + Circle + Sparkline` label came out blank on macOS 26; replaced with a guaranteed-visible SF Symbol + monospaced elapsed text.

## Install

```bash
brew install xcodegen
git clone https://github.com/nangert/TimesBar.git
cd TimesBar
make install
open /Applications/TimesBar.app
```

Or download `TimesBar-v0.2.0.zip` from this release, unzip, drag to `/Applications`. First-launch Gatekeeper warning is expected — right-click → **Open**, or:

```bash
xattr -dr com.apple.quarantine /Applications/TimesBar.app
```

## Setup

Paste a Kimai API token from your user profile (**Settings → API access**) → **Verify & save**. The app then bootstraps everything else on its own.

## Known limitations

- Kimai base URL is still hardcoded (`https://times.lipsum.services`). Editing `TimesBar/Networking/KimaiClient.swift` and rebuilding is the workaround.
- No Developer ID signing yet — Gatekeeper warns on first launch (see workaround above).
- The Stundenkonto / `-58:24h` overtime balance from your Kimai profile isn't reproducible here yet — the contract bundle's working-time data isn't in the swagger your install exposes.
